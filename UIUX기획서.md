# [UI/UX 기획서] 그린테이블 홈페이지 디자인 및 사용성 기획

## 1. 전체 디자인 콘셉트
* **콘셉트 키워드:** Eco-Chic (친환경적이면서도 세련된), Organic Modern (자연스럽고 모던한)
* **디자인 무드:** 복잡한 그래픽 요소를 배제하고 여백(White Space)을 충분히 활용하여 깔끔하고 신뢰감 있는 분위기 연출. 고화질의 신선한 음식 사진과 자연의 텍스처를 살려 건강함을 직관적으로 전달.

---

## 2. 메인 컬러와 보조 컬러
* **Main Color (메인 컬러):** 
  * `Forest Green (#2D6A4F)` : 브랜드의 핵심인 '친환경'과 '건강함'을 상징하는 깊고 차분한 녹색. CTA 버튼 및 주요 강조 텍스트에 사용.
* **Sub Color (보조 컬러):**
  * `Warm Ivory (#F9F7F3)` : 배경색으로 사용하여 눈의 피로를 줄이고, 자연스러운 오가닉 느낌을 부여.
  * `Earth Brown (#A47E53)` : 아이콘이나 보조 텍스트에 사용하여 땅과 자연의 느낌을 더함.
* **Accent Color (포인트 컬러):**
  * `Fresh Orange (#F4A261)` : 식욕을 돋우고 시선을 끄는 포인트 컬러. 'Best', 'New' 등의 뱃지나 아이콘 포인트에 제한적으로 사용.

---

## 3. 폰트 스타일
* **Headings (제목형 폰트):** `Pretendard (Bold/ExtraBold)`
  * 모던하고 가독성이 뛰어나 세련된 브랜드 이미지를 전달.
* **Body (본문형 폰트):** `Pretendard (Regular/Medium)`
  * 웹/모바일 환경에서 최적의 가독성을 자랑하며 제목과의 통일성을 유지.
* **Point (영문 포인트 폰트):** `Montserrat`
  * 영문 슬로건이나 카테고리명(Home, Menu 등)에 사용하여 트렌디한 감각 추가.

---

## 4. 메인 화면 구성 (Home)
* **Hero Section:** 화면을 꽉 채우는 신선한 샐러드/도시락 이미지와 함께 굵고 명확한 슬로건 배치. (예: "매일 만나는 자연, 건강한 한 끼"). 하단에 '메뉴 보러가기' 메인 CTA 버튼 배치.
* **Feature Section:** 3개의 심플한 라인 아이콘(친환경 용기, 당일 조리, 균형 잡힌 영양)을 가로로 배치하여 브랜드 핵심 가치를 빠르게 전달.
* **Best Menu Slider:** 사용자가 스와이프하거나 화살표를 클릭해 인기 메뉴를 넘겨볼 수 있는 캐러셀(Carousel) UI 적용.

---

## 5. 브랜드 소개 영역 구성 (Brand)
* **스플릿 레이아웃 (Split Layout):** 좌측에는 따뜻한 조명 아래 정성껏 조리하는 사진을, 우측에는 브랜드 스토리를 배치하여 시선이 자연스럽게 사진에서 글로 이동하도록 유도.
* **숫자로 보는 그린테이블:** 누적 판매량, 정기구독자 수, 플라스틱 절감량 등을 카운팅 애니메이션(시각적 효과)으로 표현하여 신뢰도 상승.

---

## 6. 메뉴 소개 카드 구성 (Menu Card)
* **이미지 영역:** 모서리가 둥근(Border-radius) 카드 형태. 메뉴의 45도 뷰 또는 탑뷰 고화질 이미지 배치.
* **정보 영역:** 메뉴명, 한 줄 설명, 가격을 깔끔하게 정렬.
* **호버(Hover) 인터랙션:** 마우스를 올렸을 때 이미지가 살짝 확대(Scale-up)되며, 반투명한 오버레이 위로 '칼로리(kcal)'와 '주요 영양성분(단백질 등)' 정보가 부드럽게 나타나도록 설계.
* **뱃지 시스템:** 좌측 상단에 'Vegan', 'Keto', 'Best' 등의 라벨을 달아 타깃 고객의 빠른 선택을 도움.

---

## 7. 친환경 가치 소개 영역 구성 (Eco-Value)
* **인포그래픽 활용:** 긴 텍스트 대신 '자연 분해되는 용기', '재활용 가능한 커트러리' 등의 요소를 일러스트나 아이콘 기반의 인포그래픽으로 구성.
* **타임라인/스텝 UI:** 식재료 수확부터 고객의 식탁에 오르기까지, 그리고 식사 후 자연으로 돌아가는 과정을 3~4단계의 스텝 UI로 시각화하여 지루하지 않게 전달.

---

## 8. 문의하기 영역 구성 (Inquiry)
* **플로팅 라벨(Floating Label) 폼:** 입력창에 텍스트를 입력하면 안내 문구(Placeholder)가 위로 작게 올라가는 UI를 적용하여 폼을 깔끔하게 유지.
* **조건부 입력 UI:** 문의 유형(단체주문 / 정기배송)을 선택하는 라디오 버튼을 최상단에 배치하고, 선택에 따라 필요한 입력 항목만 노출되도록 하여 사용자 피로도를 최소화.
* **명확한 피드백:** 필수 항목 미입력 시 붉은색 테두리와 툴팁으로 직관적인 에러 메시지 제공. 전송 완료 시 팝업 대신 화면 전환을 통해 명확한 완료 메시지와 체크 애니메이션 제공.

---

## 9. 모바일 반응형 구성 (Mobile Responsive)
* **네비게이션 바:** 모바일에서는 햄버거 메뉴(☰)를 우측 상단에 배치하고, 열었을 때 전체 화면을 덮는 풀스크린(Full-screen) 메뉴로 구성하여 터치 영역을 넓게 확보.
* **Sticky Bottom 버튼:** 모바일 화면 하단에 '주문 문의하기' 플로팅 버튼을 항상 고정시켜두어, 스크롤을 내리다가도 언제든 전환(Conversion)이 일어날 수 있도록 유도.
* **카드 레이아웃:** PC에서 3~4단이었던 메뉴 카드는 모바일에서 1단(Full width) 또는 가로 스와이프(Horizontal Scroll) 형태로 변경하여 세로 스크롤 길이를 최적화.

---

## 10. 사용자 경험을 높이기 위한 요소 (UX Enhancements)
* **마이크로 인터랙션:** 버튼에 마우스를 올리거나 클릭할 때 컬러가 부드럽게 변하거나 둥글게 퍼지는 리플(Ripple) 효과를 주어 앱을 사용하는 듯한 만족감 제공.
* **스켈레톤 UI (Skeleton Loading):** 메뉴 이미지가 로딩되는 동안 빈 화면 대신 회색 레이아웃 박스가 먼저 보이는 스켈레톤 UI를 적용하여 체감 로딩 속도를 단축.
* **퀵 스크롤 (Scroll to Top):** 페이지 하단으로 내려갔을 때 우측 하단에 '위로 가기' 화살표 버튼을 배치하여 탐색의 편의성 증대.
* **접근성(Accessibility) 고려:** 모든 텍스트와 배경 컬러 간의 명도 대비를 충분히 확보하고, 이미지에는 `alt` 태그를 적용하여 스크린 리더 사용자를 배려.
